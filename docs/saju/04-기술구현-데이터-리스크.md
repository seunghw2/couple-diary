# 04 · 기술 구현·데이터·리스크·정책

## 0. 재사용 기반 (코드에서 확인)
월드컵의 **정적 카탈로그 + per-user 결과 행 + 커플 비교** 3단 구조를 그대로 재사용 가능:
- `worldcup/WorldcupCatalog.java` — 콘텐츠를 코드 상수로 고정("추가는 끝에만, id 유지" 규칙). 사주 문구도 동일 방식.
- `worldcup/WorldcupResult.java` — `@ManyToOne User author` + `@ManyToOne Couple couple` + 문자열 인코딩 페이로드 + createdAt, 복합 인덱스. **사주 프로파일 엔티티가 복제할 골격.**
- `worldcup/WorldcupResultRepository.java` — 최신 1건 조회 + `deleteByCouple_Id`/`deleteByAuthor_Id`. **삭제 cascade가 이미 `UserService`(라인 281)에 등록** → 사주 리포지토리도 한 줄만 추가하면 계정삭제 정합성 유지.
- `WorldcupService.java` — `coupleService.requireCouple` → `memberOf/partnerOf` → 둘 다 있으면 비교. 완주 알림까지 처리. **"둘 다 있을 때만 궁합"이 동일 패턴.**

**핵심 제약:** `User.birthday`는 `LocalDate`(시각 없음, `User.java:48`). 온보딩·DatePickerSheet도 날짜만 → **생시를 별도 수집해야 함.** `ddl-auto: update`라 엔티티 추가 시 스키마 자동 반영(마이그레이션 불필요).

## 1. 계산 위치: 백엔드(Java) 권장
- **결정론적 → 1회 계산 후 캐시** 자연스러움(worldcup 결과 저장 철학과 동일).
- 절기 테이블/만세력을 **서버 한 곳**에만 → 앱 번들 안 늘고, 데이터·로직을 서버 배포로만 갱신(심사 재제출 불필요).
- 커플 궁합은 **두 사람 데이터** 필요 → 서버가 모아 계산(상대 생년월일을 클라에 노출 안 함, 프라이버시).
- **권장 형태**: 최초 조회 시 계산 → `saju_profile` 캐시 → 이후 캐시 반환. 입력 수정 시 무효화.

## 2. 절기/만세력 데이터
| 기둥 | 필요 데이터 | 난이도 |
|---|---|---|
| **일주** | 없음(공식) | 쉬움 |
| **연주** | 입춘 경계 | 중 |
| **월주** | 12절 경계 시각 | 중~상 |
| **시주** | 생시+일간 | 쉬움 |

### 일주 — 데이터 없이 정확 ✅
JDN(=`LocalDate.toEpochDay()`)로 60갑자 순환. `(toEpochDay() - base) mod 60`. 외부 데이터·라이브러리 불필요, 만세력과 검증 가능. 시주도 일간 나오면 표로 계산.

### 월주·연주 — 절기 경계, 3옵션
- **(a) 근사 알고리즘**(평기법/저정밀 태양황경): 데이터 0, 정확도 최대 ±1~2일. 절기 경계 출생자 오차. 재미용 허용, 헤비 유저는 눈치챌 수 있음.
- **(b) 연도별 절기 테이블 내장 (권장) ✅**: KASI 정기법 절입 시각을 **100년×12절 ≈ 1,200행** JSON/CSV(수십 KB) 서버 번들. 정확도=국내 만세력 완전 일치. KASI 공공데이터(공공누리, 출처표기 권장). 유지보수: 몇 년에 한 번 연장.
- **(c) 오픈소스 라이브러리/천문 계산**: `commons-suncalc`·Meeus 기반으로 태양황경→절기(분 단위). 사주 특화 검증된 JVM 라이브러리는 드물어 직접 검증 부담.

**권장:** MVP는 **(b) KASI 절기 테이블 내장**. 런타임 외부 API 의존은 지양(장애·레이트리밋). 배포 시 KASI Open API 1회 호출로 테이블 시딩하는 하이브리드도 가능.

## 3. 데이터 모델

### `saju_profile` (per user, 캐시)
```
id, user(@ManyToOne unique), couple(@ManyToOne),
birth_date(LocalDate), birth_time_slot(enum/int nullable, 12지시 or 모름),
calendar_type(SOLAR/LUNAR),
pillars(String, "갑자,을축,병인,정묘" 인코딩),
day_master(일간 甲~癸), element_dist(String, "목2화1토1금3수1"),
zodiac(띠), created_at
```
- worldcup처럼 **문자열 인코딩**으로 컬럼 폭발 방지. 생시 "모름" 분기 필수.

### `saju_couple_result` (per couple, 궁합)
- worldcup `CompareView`처럼 **둘 다 프로파일 있을 때만** 생성. 두 프로파일에서 결정론적으로 나오므로 **저장 없이 즉석 계산 가능**(캐시는 선택).

### worldcup_result와 비교
- 공통: author/couple FK, createdAt, 문자열 페이로드, "둘 다 있을 때 비교", 삭제 cascade 훅.
- 차이: worldcup은 여러 번 플레이(여러 행), 사주는 **입력당 결과 1건** → `saju_profile`은 유저당 1행(upsert), 재계산은 입력 변경 시만.

### 민감정보
- 생년월일+생시 = 개인정보(생일은 이미 있음, 생시는 신규 준민감). 처리방침에 항목·목적 명시. 입력 불변 → 캐시. **"계산값 캐시"와 "표시 시 문구 매핑"을 분리**하면 문구 수정이 원국에 영향 없음.

## 4. 고정 템플릿 파이프라인

### 저장 위치: **상수 클래스 or 리소스 JSON** (DB 아님)
worldcup 카탈로그와 동일 — 문구는 읽기전용·git 리뷰 대상. DB는 시딩·마이그레이션 부담. MVP는 `SajuCatalog` 상수, 커지면 `resources/saju/*.json`.

### 필요 개수 — 축 분리로 폭발 억제
- 일간 10 + 오행 코멘트 10~15 + 띠 12 + 카테고리 요약 몇 개 ≈ 개인 40~50
- 궁합 등급×카테고리 15~20 + 오행 상성 코멘트 10~15 ≈ 30~40
- **MVP 총 60~120개**. 곱집합(수천 개) 대신 **축 분리·재조합**.

### 유지보수
- "id 고정, 추가는 끝에만" 규칙. **계산(원국)과 문구(표시) 분리**. 톤은 긍정·가벼움.

## 5. 앱스토어/법적·정책 리스크

### Apple
- **핵심 리스크 4.3(Spam, 포화 카테고리)**: 주 기능이 운세인 앱 반려 사례 다수. **완화: 이 앱 주 기능은 다이어리, 사주는 서브 미니게임**(설정 진입). 4.3는 primary function을 보므로 서브면 위험 낮음. **사주를 전면화·단독앱화하면 위험↑.**
- 면책 문구가 통과를 보장하진 않지만(반려는 신뢰성보다 포화) 소비자 오인·법적 리스크 완화용으로 권장.
- 연령등급: 도박·유료 예언 없으면 낮게 유지.

### Google Play
- 더 관대. 무료 재미 기능이면 리스크 낮음. "정확한 예언" 유료 판매만 주의.

### 한국 개인정보/문화
- 생년월일시 수집 → 처리방침에 항목·목적·보관기간. 생시 추가 시 방침 업데이트.
- 단정적·부정적 예언(건강·수명·사고) 지양, "재미로 보는, 실제와 무관" 면책.
- 한국에서 사주·궁합은 커플 맥락에 매우 친숙(참여 동기) → 다이어리와 궁합 좋음. 톤은 "운명 결정론" 아닌 "성향 놀이".

### 완화 체크리스트
- [ ] 사주를 **서브 기능** 배치, 소개·스크린샷 주 기능은 다이어리
- [ ] 결과·최초 진입에 **면책 표기**
- [ ] 문구 톤 긍정·가벼움, 건강/수명/사고 예언 배제
- [ ] 처리방침에 생년월일·생시 수집 추가, 생시 **선택 입력**
- [ ] 유료로 "정확도" 판매 금지
- [ ] 절기 데이터 KASI 사용 시 출처 표기
- [ ] 계정삭제 시 사주 프로파일 정리(`UserService` 삭제 훅에 리포지토리 추가)

## 6. 단계적 로드맵

### MVP
1. **계산**: 일주·시주 = JDN/`toEpochDay` 공식(데이터 0). 월주·연주 = **KASI 절기 테이블(100년·12절) 내장**. 오행·일간·띠 도출.
2. **엔티티**: `saju_profile`(유저당 1건) + 삭제 cascade 훅. 궁합은 저장 없이 즉석 계산.
3. **생시 입력**: 12지시 + "모름". 사주 진입 시 입력(생시 미상 분기). 음력은 후속(MVP 양력만).
4. **문구**: `SajuCatalog` 상수 60~120개, 긍정 톤 + 면책.
5. **API/프론트**: `/api/saju` — worldcup 미러. 화면은 worldcup 스타일 재사용.
6. **정책**: 서브 배치 + 면책 + 처리방침 업데이트.

### 확장
- 음력 생일(KASI 음↔양 변환), 야자시/조자시 정밀화
- 궁합 캐시 + 완주/궁합 알림(worldcup 패턴)
- 문구 다양화, JSON 리소스 분리
- "오늘의 운세"(일진 lookup)
- 절기 테이블 연장/자동 시딩
