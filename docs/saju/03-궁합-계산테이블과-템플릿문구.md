### A-1. 오행 상생/상극/같음 관계 판정

오행 관계는 "일간 A오행 → 일간 B오행" **방향이 있는** 관계다.
정의(전통):
- **상생(相生)** 목→화→토→금→수→목 : A가 B를 낳아줌.
- **상극(相剋)** 목→토→수→화→금→목 : A가 B를 이김.

판정 규칙(오행 a에서 b로):

```
같음(SAME)        : a == b
상생-순방향(SHENG_FWD): b == (a + 1) % 5           // a가 b를 생함 (내가 상대를 북돋움)
상생-역방향(SHENG_REV): a == (b + 1) % 5           // b가 a를 생함 (상대가 나를 북돋움)
상극-순방향(KE_FWD)   : b == (a + 2) % 5           // a가 b를 극함 (내가 상대를 누름)
상극-역방향(KE_REV)   : a == (b + 2) % 5           // b가 a를 극함 (상대가 나를 누름)
```

(오행 index가 木0 火1 土2 金3 水4 순서이므로 `+1`이 상생 다음 오행, `+2`가 상극 대상이 되도록 정렬되어 있음. 검산: 木0→+1=火1 상생 OK, 木0→+2=土2 상극 OK.)

**관계 유형 enum (5종)**:

```java
enum ElemRel { SAME, SHENG_FWD, SHENG_REV, KE_FWD, KE_REV }

static ElemRel elemRel(int a, int b){
    if(a==b) return ElemRel.SAME;
    if((a+1)%5==b) return ElemRel.SHENG_FWD;
    if((b+1)%5==a) return ElemRel.SHENG_REV;
    if((a+2)%5==b) return ElemRel.KE_FWD;
    return ElemRel.KE_REV;  // (b+2)%5==a 가 유일하게 남음
}
```

**점수화 시 "상생/상극"으로 뭉쳐 쓸 때의 정의**:
- 상생계(SHENG) = SHENG_FWD 또는 SHENG_REV
- 상극계(KE) = KE_FWD 또는 KE_REV
- 같음 = SAME

> 상극은 원설계대로 **무조건 감점 아님**. "과다 제어" 보너스는 A-5에서 별도 처리. 문구에서는 상극을 절대 "상극"이라 부르지 않고 "밀당/자극 케미"로만 노출.

### A-4. 오행 분포 & 상보(빈 오행 보완) 판정

#### A-4-1. 개인 오행 분포 벡터

- 각 글자의 오행은 A-0의 GAN_ELEM / JI_ELEM 사용. 지장간(숨은 천간)은 MVP에서 **미사용**(결정론·단순성 우선). 필요 시 후속.

#### A-4-2. 부족 오행(약오행) 집합

```java

    var s = new java.util.HashSet<Integer>();
    for(int e=0;e<5;e++) if(dist[e]==0) s.add(e);
    return s;   // 전부 채워져 있으면 빈 set
}
```

#### A-4-3. 상보(빈 오행 보완) 점수 — sangboScore (0..1)

핵심 따뜻한 서사. "내게 없는 오행을 상대가 넉넉히 갖고 있는가"를 양방향으로 본다.

```
lackA = A의 부족 오행 집합, lackB = B의 부족 오행 집합
coveredAbyB = { e in lackA : B.dist[e] >= 2 }   // 상대가 2개 이상이면 '넉넉히 보완'으로 인정
coveredBbyA = { e in lackB : A.dist[e] >= 2 }
totalLack   = lackA.size + lackB.size
covered     = coveredAbyB.size + coveredBbyA.size

sangboScore = (totalLack == 0) ? 0.6           // 둘 다 오행이 고르면 자체로 안정 → 중상값
                               : covered / totalLack   // 0..1
```

- 임계값 `>=2`는 "우연히 1개 겹침"을 배제하기 위함(튜닝 상수 `SANGBO_MIN=2`).
- `totalLack==0`(양쪽 다 오행 완비)일 땐 보완할 게 없지만 그 자체가 균형이므로 **0.6 기본값** 부여(감점 방지).

#### A-4-4. 합산 오행 균형도 — balanceScore (0..1)

두 사람 dist를 합쳐(`combined[e]=A.dist[e]+B.dist[e]`) 얼마나 고른지.

```
mean = sum(combined)/5.0
variance = Σ (combined[e]-mean)^2 / 5.0
maxVarianceApprox = mean*mean*4   // (한 오행에 전부 몰렸을 때의 대략 상한; 0 나눗셈 방지용 근사)
balanceScore = 1 - clamp(variance / maxVarianceApprox, 0, 1)   // 고를수록 1
// sum==0 이면(이론상 없음) balanceScore=0.5 로 폴백
```

#### A-4-5. 음양 균형도 — yinYangScore (0..1)

두 사람의 모든 글자 음양 합을 본다. 한쪽으로 심하게 쏠릴수록 낮게, 섞일수록 높게.

```
yySum = Σ(A글자 음양) + Σ(B글자 음양)   // 각 글자 +1/-1
n     = A글자수 + B글자수
imbalance = |yySum| / n                 // 0(완전균형)..1(완전쏠림)
yinYangScore = 1 - imbalance            // 1 = 음양 조화
```

### A-5. "상극의 과다 제어" 긍정 재해석 판정 (⑤ 성장 카테고리 전용)

상극을 감점이 아니라 **+로 재해석**하는 조건.
"A일간 오행이 B에게 과다한 오행을 눌러 균형을 맞춰주는가?"

```
일간 상극 관계일 때(elemRel(dayElemA,dayElemB) in {KE_FWD, KE_REV}):
  극당하는 쪽 오행 = KE_FWD 이면 dayElemB, KE_REV 이면 dayElemA
  '과다'기준: 상대(또는 본인) 합산 관점에서 그 오행 count가 combined에서 최다이거나 mean*1.5 이상
  → 조건 충족 시 controlBonus = true  (⑤에 +CONTROL)
  → 미충족 시 상극은 ⑤에서 감점하지 않고 0 (다른 카테고리에서만 소폭 반영)
```

```java
static boolean isOverloadControl(int dayElemA,int dayElemB,int[] combined){
    ElemRel r = elemRel(dayElemA,dayElemB);
    if(r!=ElemRel.KE_FWD && r!=ElemRel.KE_REV) return false;
    int suppressed = (r==ElemRel.KE_FWD)? dayElemB : dayElemA;
    double mean = 0; for(int c:combined) mean+=c; mean/=5.0;
    int maxE=0; for(int e=1;e<5;e++) if(combined[e]>combined[maxE]) maxE=e;
    return combined[suppressed] >= Math.max(mean*1.5, 1) &&
           (suppressed==maxE || combined[suppressed]>=mean*1.5);
}
```

### A-6. 5개 카테고리 원점수 계산식 (각 0~100)

공통: 각 카테고리는 **가산요소 합 - 감점요소**를 구한 뒤 `clamp(0,100)`.
아래 가중치는 확정값(코드 상수로 그대로 옮길 것). 각 카테고리에 **베이스 점수**를 둬서 요소가 하나도 안 걸려도 바닥이 너무 낮지 않게 한다(커플 앱 톤).

> 자리 규약: `day`=일간(천간), `dayJi`=일지, `monJi`=월지, `yearJi`=년지(띠), `hourJi`=시지(있을 때만). 없으면 해당 항 0.

#### ① 첫끌림 케미 (CHEMI) — base 30

| 요소 | 조건 | 점수 |
|---|---|---|
| 천간합 | isGanHap(dayA,dayB) | +40 |
| 삼합(년지) | isSamhap(yearJiA,yearJiB) | +25 |
| 육합(년지) | isYukhap(yearJiA,yearJiB) | +15 |
| 일간 상생 | elemRel(dayElemA,dayElemB) in SHENG | +15 |
| 일간 같음 | SAME | +8 |
| 음양 상보(일간) | !sameYinYang(dayA,dayB) | +10 |
| 천간충 | isGanChung(dayA,dayB) | −12 (순화, 문구는 "자극 케미") |

```
CHEMI = clamp(30 + (ganHap?40:0) + (samhapYear?25:0) + (yukhapYear?15:0)
             + (shengDay?15: sameDay?8:0) + (yinYangComplementDay?10:0)
             - (ganChung?12:0), 0, 100)
```

#### ② 대화·소통 (TALK) — base 35

| 요소 | 조건 | 점수 |
|---|---|---|
| 일간 상생방향 | SHENG_FWD 또는 SHENG_REV | +30 |
| 일간 같음 | SAME | +18 |
| 월지 삼합 | isSamhap(monJiA,monJiB) | +18 |
| 월지 육합 | isYukhap(monJiA,monJiB) | +14 |
| 오행분포 유사 | distSimilarity>=0.6 (아래식) | +20 |
| 월지 충 | isJiChung(monJiA,monJiB) | −18 |

`distSimilarity = 1 - Σ|Aratio[e]-Bratio[e]| / 2`  (Aratio=dist를 합=1로 정규화; 코사인 대신 L1, 0..1)

```
TALK = clamp(35 + (shengDay?30: sameDay?18:0) + (samhapMon?18: yukhapMon?14:0)
            + (distSim>=0.6?20:0) - (chungMon?18:0), 0,100)
```

#### ③ 애정 표현 (AFFECTION) — base 32

| 요소 | 조건 | 점수 |
|---|---|---|
| 일지 육합 | isYukhap(dayJiA,dayJiB) | +35 |
| 일지 삼합 | isSamhap(dayJiA,dayJiB) | +25 |
| 화(火) 총량 | combined[火]>=3 | +15 |
| 음양 조합 | yinYangScore>=0.5 | +15 |
| 일지 충 | isJiChung(dayJiA,dayJiB) | −15 (순화) |
| (원진 플래그) | 감점 없음 · 문구 트리거만 | 0 |

```
AFFECTION = clamp(32 + (yukhapDay?35:0) + (samhapDay?25:0) + (fire>=3?15:0)
                 + (yinYangScore>=0.5?15:0) - (chungDay?15:0), 0,100)
```

#### ④ 안정감·장기 (STABILITY) — base 30

| 요소 | 조건 | 점수 |
|---|---|---|
| 일지 육합 | isYukhap(dayJiA,dayJiB) | +30 |
| 오행 상보 | sangboScore | +round(30*sangboScore) |
| 전체 균형 | balanceScore | +round(20*balanceScore) |
| 일지 삼합 | isSamhap(dayJiA,dayJiB) | +12 |
| 일지 충 | isJiChung(dayJiA,dayJiB) | −20 (순화) |

```
STABILITY = clamp(30 + (yukhapDay?30:0) + round(30*sangboScore) + round(20*balanceScore)
                 + (samhapDay?12:0) - (chungDay?20:0), 0,100)
```

#### ⑤ 성장·밸런스 (GROWTH) — base 40 (안전판: 낮은 총점을 "성장형"으로 완충)

| 요소 | 조건 | 점수 |
|---|---|---|
| 상극 과다제어 | isOverloadControl(...) | +30 (긍정 재해석) |
| 음양 균형 | yinYangScore | +round(25*yinYangScore) |
| 상보성 | sangboScore | +round(25*sangboScore) |
| 극단 쏠림 | balanceScore<0.3 | −15 |

```
GROWTH = clamp(40 + (overloadControl?30:0) + round(25*yinYangScore)
              + round(25*sangboScore) - (balanceScore<0.3?15:0), 0,100)
```

> 설계상 ⑤는 다른 곳에서 감점되는 상극/충 커플에게 오히려 높은 점수를 줘, 결과 카드에 항상 "강점 카드"가 하나는 남도록 하는 장치다.

### A-7. 종합 점수 · 리매핑 · 등급 · 배지

#### A-7-1. 가중 합 (원점수 0~100)

```
WEIGHTS: CHEMI 0.25, TALK 0.20, AFFECTION 0.20, STABILITY 0.20, GROWTH 0.15
rawTotal = CHEMI*0.25 + TALK*0.20 + AFFECTION*0.20 + STABILITY*0.20 + GROWTH*0.15
// rawTotal ∈ [0,100]
```

#### A-7-2. 60~99% 리매핑 (기분 좋게, 최고 99)

실전 rawTotal은 대략 35~85 중앙에 몰린다. 관측 하한/상한을 잡아 **60~99**로 선형 확장.

```
RAW_FLOOR = 30.0   // 이 이하도 60%로 바닥 보장
RAW_CEIL  = 90.0   // 이 이상은 99%로 상한
norm = clamp((rawTotal - RAW_FLOOR) / (RAW_CEIL - RAW_FLOOR), 0, 1)   // 0..1
displayPercent = round(60 + norm * 39)   // 60..99
// 안전: 결과가 100 나오지 않도록 39 사용(최고 99). 최저 60 보장.
```

**결정론 보장**: rawTotal은 순수 계산, 리매핑도 순수 함수 → 같은 두 생일이면 항상 동일 displayPercent.

#### A-7-3. 등급 컷 (문구 선택용 · rawTotal 기준 아님, **카테고리별 원점수** 기준)

각 카테고리 문구는 그 **카테고리 원점수(0~100)**로 등급을 고른다:

```
등급(catScore):
  상(HIGH): 67 <= score <= 100
  중(MID):  34 <= score <= 66
  하(LOW):   0 <= score <= 33
```

종합 구간(displayPercent 기준)으로 종합 문구 선택:

```
95 <= p          → TOTAL_95
85 <= p < 95     → TOTAL_85
75 <= p < 85     → TOTAL_75
65 <= p < 75     → TOTAL_65
60 <= p < 65     → TOTAL_60
```

#### A-7-5. 카테고리별 "일간 오행 관계" 코멘트 트리거 (문구용)

`elemRel(dayElemA,dayElemB)` 결과 → 한줄 코멘트 그룹 키:
```
SAME      → REL_SAME_*
SHENG_*   → REL_SHENG_*   (방향 무시하고 동일 그룹)

```
상보 하이라이트는 `sangboScore>=0.5`일 때 `REL_SANGBO_*` 추가 노출.

#### A-7-6. 문구 선택의 결정론 (랜덤 금지)

한 등급/구간에 문구가 2~3개면, **인덱스를 두 생일 해시로 고정**한다(같은 커플=항상 같은 문구, 그러나 커플마다 다양).

```
seed = (yyyymmddA * 100000000L + yyyymmddB)   // 두 생일 정수 결합(작은쪽*큰쪽 순으로 정렬해 대칭 보장)
idx  = (int)(Math.floorMod(seed, templateCount))
```
> 대칭성: A,B 순서 바뀌어도 같은 결과 나오게 `min,max`로 정렬 후 결합.
