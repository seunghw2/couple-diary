# love today · 사주보기 기능 기획

커플 다이어리 앱 **love today**에 **월드컵 미니게임처럼 가벼운 사주보기** 기능을 넣기 위한 분석·기획 문서 모음입니다. 서브에이전트 4개의 병렬 분석을 종합했습니다.

## 확정된 방향(사용자 결정)
- **초점**: 개인 사주 + 커플 궁합 **둘 다 제공**
- **생시(태어난 시각)**: **선택 입력** (모르면 생년월일만으로도 결과)
- **결과 콘텐츠**: **고정 템플릿** (비용 0·오프라인·심사 안전)
- **톤**: 정통 역술 X, **월드컵처럼 재미 위주**의 서브 미니게임
- **절기 정확도**: **처음부터 KASI 정확 절기 테이블 내장** (시중 만세력과 일치)
- **오늘의 운세**: **포함** (날짜별로 바뀌는 데일리 재방문 요소)
- **음력 생일**: **지원** (양력 변환 처리)
- **기능 이름**: **"우리 사주 궁합"**

> 참고: 위 결정으로 1차 범위가 최소 MVP보다 커졌음 — 절기 테이블 확보, 음력→양력 변환, 오늘의 운세 로직이 처음부터 포함된다. 품질은 높아지고 착수 준비 작업(특히 절기·음양력 데이터)이 늘어난다.

## 한 줄 결론
> 기존 **월드컵 기능의 구조(설정 진입 → 배지 → 허브 → 좌우 2열 커플 비교 → 알림 딥링크)를 그대로 복제**하고, 콘텐츠만 "토너먼트"에서 "생년월일 → 결정론적 사주 계산 + 고정 문구"로 바꾸면 된다. 계산은 백엔드에서, 일주·시주는 데이터 없이 정확 계산되고 월주만 절기 테이블이 필요하다.

## 핵심 요약(각 분석에서)
- **계산(01)**: 일주·시주는 `LocalDate.toEpochDay() mod 60`류 공식으로 **데이터 없이 100% 정확**. 월주·년주는 절기 경계가 필요 → **KASI 정기법 절기 테이블(100년·12절 ≈ 1,200행) 내장** 권장. 십신·용신은 빼고 **일간 캐릭터 + 띠 + 오행 분포** 중심으로 단순화.
- **궁합(02)**: 축은 (일간 천간합/충) + (오행 상생·상극·상보) + (지지 삼합/육합/충) + (띠·음양). 무서운 살(煞)·형·파·해는 빼거나 "츤데레 포인트"로 순화. 5개 카테고리(첫끌림·대화·애정·안정감·성장)로 0~100 원점수 → **재미용 60~99% 리매핑**. 문구 65~70개면 충분.
- **UX(03)**: 설정 "바로가기"에 월드컵 아래 **"우리 사주 궁합"** 행(배지 포함). 허브 `/saju` → `내 사주`/`우리 궁합` 2카드. 상대 생일 없으면 "생일 요청" 알림. **결과 저장 엔티티 불필요**(계산이 결정론적) — 유일한 신규 저장은 선택 입력 생시.
- **기술(04)**: 백엔드 계산+캐시 권장. `saju_profile`(유저당 1행) + 궁합은 즉석 계산. 문구는 `SajuCatalog` 상수. 앱스토어 4.3(운세 포화) 리스크는 **서브 미니게임으로 배치 + 면책 문구**로 회피. 계정삭제 cascade 훅 1줄 추가.
