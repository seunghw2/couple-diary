# 01 · 사주 개념과 계산 로직

> 목적: 커플 다이어리 앱의 '월드컵 미니게임'급 가벼운 사주보기. 생년월일 필수 / 생시 선택 / 고정 템플릿 결과 / 오프라인 계산.

## 1. 사주 기본 구조

### 사주(四柱) = 네 개의 기둥
- **사주팔자**: 태어난 **연·월·일·시**를 각각 하나의 '기둥(柱)'으로 세운 것. 기둥마다 **천간 1글자 + 지지 1글자 = 2글자**, 총 **4기둥 × 2 = 8글자(八字)**.
  - **년주(年柱)**: 태어난 해 — 조상·초년·사회적 나
  - **월주(月柱)**: 태어난 달(절기 기준) — 부모·청년·환경/사회성
  - **일주(日柱)**: 태어난 날 — **본인**(일간)과 배우자(일지)
  - **시주(時柱)**: 태어난 시각 — 자식·말년·내면
- 앱에서 생시를 안 받으면 **시주만 비고 나머지 3주(六字)는 항상 계산 가능**.

### 천간(天干) 10개
- 甲(갑) 乙(을) 丙(병) 丁(정) 戊(무) 己(기) 庚(경) 辛(신) 壬(임) 癸(계)
- 순서대로 **오행 × 음양** 조합: 甲乙=목, 丙丁=화, 戊己=토, 庚辛=금, 壬癸=수. (짝수 번째가 음)

### 지지(地支) 12개 = 띠(十二支)
- 子(쥐) 丑(소) 寅(호랑이) 卯(토끼) 辰(용) 巳(뱀) 午(말) 未(양) 申(원숭이) 酉(닭) 戌(개) 亥(돼지)
- 각 지지도 오행/음양/계절을 가짐(寅卯=봄/목, 巳午=여름/화 …). **년지 = 띠**.

### 60갑자(六十甲子)
- 천간 10 × 지지 12의 **최소공배수 60**개 조합. 甲子 → 乙丑 → … → 癸亥 → 다시 甲子.
- **핵심 성질: 정확히 60을 주기로 순환** → 소프트웨어에서 **나머지 연산(mod 60)** 으로 결정론적 계산 가능. 이게 사주 구현의 수학적 기반.

### 음양(陰陽)·오행(五行)
- **오행**: 木(목)·火(화)·土(토)·金(금)·水(수). 각 글자는 반드시 오행 하나에 속함.
  - **상생**: 목→화→토→금→수→목 (낳아줌)
  - **상극**: 목→토→수→화→금→목 (억누름)
- **음양**: 각 글자에 +/− 부여. 8글자의 음양·오행 분포가 성향 해석의 재료.

### 일간(日干, Day Master)이 왜 핵심인가
- **일간 = 일주의 천간 = '나 자신'**. 나머지 7글자는 전부 **일간을 기준으로** 관계가 정의됨.
- 사주 해석은 "이 사람은 무슨 오행인가?"에서 시작 → 곧 **일간이 무엇이냐**가 캐릭터의 근본. (예: 甲木 일간 = 큰 나무, 곧게 뻗는 리더형 / 癸水 일간 = 이슬·비, 섬세·유연 등 — 가벼운 앱의 '캐릭터 카드'로 직결)
- **오행 균형도** 일간을 기준으로 판단.

### 십신·지장간 (앱에선 간단히만)
- **십신(十神)**: 일간과 다른 글자의 관계를 10가지로 이름 붙인 것. 정통 해석의 핵심이지만 **로직·표가 복잡** → 가벼운 앱에선 생략 권장.
- **지장간(支藏干)**: 지지 하나 안에 천간 2~3개가 숨어있다는 개념. 정밀 오행 분포에 쓰이나 **표 필요** → 가벼운 버전에선 지지의 대표 오행만 써도 충분.

## 2. 생년월일(+선택 생시)로 결정론적 계산

### 2-1. 일주(일간·일지) — 가장 쉽고 정확 ✅
- **원리**: 60갑자 일진은 하루에 1씩, 60일 주기로 순환(요일과 같은 성질).
- **방법**: 양력 날짜 → **율리우스적일수(JDN)** → `index = (JDN + 오프셋) mod 60`.
  - **앵커 방식이 안전**: 검증된 기준일 하나를 박아두고 일수 차이를 mod 60.
  - 검증된 앵커: **2019-01-27 = 甲子(index 0)**, JDN=2458511. → `dayIndex = (JDN - 2458511) mod 60` (음수는 +60 보정).
  - **Java에선 `LocalDate.toEpochDay()`(1970-01-01 기준)가 JDN 대체** → `(toEpochDay() - base) mod 60` 로 끝. 외부 데이터·라이브러리 불필요.
- **난이도**: ★☆☆ / 오프라인 100% 정확.

### 2-2. 월주 — 절기(절입) 필요
- 사주의 '달'은 음력도 양력도 아닌 **24절기 중 12개 '절(節)'** 로 나뉜다(입춘·경칩·청명…). **절입 시각(시·분)** 까지 따진다.
- **월지**: 절기로 달이 정해지면 고정(입춘~경칩=寅월…). ← 표 하나면 끝.
- **월간**: **월두법(月頭法)** — 년간에 따라 그 해 寅월 천간이 정해지고 순행. 간단한 규칙식.
- **절기 데이터 트레이드오프**:
  - 정확히 하려면 **연도별 절기 절입 datetime 테이블 내장**(표준 해법).
  - 근사도 가능(절기는 매년 대략 고정 양력 날짜 ±1~2일) — 경계 며칠 출생자는 틀릴 수 있음.

### 2-3. 년주 — 입춘 경계 처리
- 사주의 '해'는 1월 1일도 설날도 아닌 **입춘(立春)** 에 바뀐다. 2월 초 출생자 주의.
- 년간지는 `(해 - 4) mod 60`류 식(甲子=서기 4년). 경계 판정만 입춘 절입시각에 의존(2-2 테이블 공유).

### 2-4. 시주 — 생시 + 일간
- **시지**: 하루를 12지지로 2시간씩(子시 23:00–01:00, …).
- **시간**: **시두법(時頭法)** — 일간으로 그날 子시 천간이 정해지고 순행.
- **생시 모를 때**: 시주 전체 생략, 6글자(년·월·일주)만 사용 → 앱에서 "생시 알면 더 정확" 안내로 자연 처리.

### 2-5. 엣지케이스
- **양력/음력**: 내부는 전부 양력(JDN) 기준. 음력 입력 지원하려면 양력 변환 테이블 필요(윤달 포함). **입력을 양력으로만 받으면 이 복잡도 제거**(가벼운 앱 권장).
- **타임존**: 전부 **KST(UTC+9)** 통일. 절기 테이블도 KST로.
- **진태양시 보정**: 정통은 경도 보정(약 -30분). 가벼운 앱은 생략 권장.
- **서머타임**: 과거 일부 연도(1948–60 일부, 1987–88). 가벼운 앱은 무시 가능.
- **야자시(23:00–24:00)**: 유파마다 다름 → **하나로 고정**(일관성이 정확도 논쟁보다 중요).

## 3. '가볍게' 버전 권장안

| 요소 | 권장 | 이유 |
|---|---|---|
| **일주(일간·일지)** | **정확 계산**(JDN mod 60) | 쉽고 100% 정확. 앱의 메인 캐릭터(일간). |
| **띠(년지)** | **정확**(입춘 경계는 근사/테이블) | 누구나 아는 요소, 몰입 강함. |
| **오행 분포/균형** | **정확 계산**(글자 오행 카운트) | "불이 강하고 물이 없어요" 식 시각화 재미. |
| **월주/시주** | **선택적**·있으면 계산 | 정밀도↑, 없어도 카드 성립. |
| **십신·지장간·용신** | **생략** | 복잡·해석 난이도↑, 가벼움 훼손. |
| **음력변환·진태양시·서머타임** | **생략/근사** | 양력만 받고 보정 빼면 복잡도 급감. |

### 권장 최소 구성
1. **일간 캐릭터 카드**: 10천간 → 10종 캐릭터("당신은 甲木 — 하늘로 뻗는 큰 나무형"). 고정 템플릿 10개.
2. **띠(년지)**: 12종. 커플 궁합에 바로 활용.
3. **오행 막대/레이더**: 목·화·토·금·수 개수 → 밸런스 그래프 + "부족한 오행" 한 줄.
4. **커플 상생상극 한 줄**: 두 일간 오행 관계("水生木 — 서로 키워주는 조합"). 5×5 표 하나.

## 4. 정확도 vs 복잡도 (3단계)

- 🟢 **간단(권장)**: 일주(정확) + 띠 + 오행 카운트. 절기는 근사/최소. 오프라인, ★☆☆. 일주 100%, 띠·월지 95%+.
- 🟡 **중간**: + 연도별 절기 테이블(JSON 번들)로 월주·년주 정확. 오프라인, ★★☆. 사실상 만세력 일치.
- 🔴 **정통**: 천문 실시간 + 진태양시·서머타임·야자시 + 십신·용신·대운. ★★★. 가벼움과 충돌 → 비권장.

### 최종 권장
- **1차 출시**: 🟢 간단 — 양력 생년월일 필수 + 생시 선택, 일주(정확)·띠·오행 분포, 결과는 일간 캐릭터 + 띠 + 오행 그래프 + 커플 상생상극 한 줄.
- **여유 되면**: 절기 테이블만 추가해 🟡 월주 정확도(가성비 최고).
- **핵심 원칙**: 미니게임에선 정확성보다 **재현성·재미**. 야자시·시주 경계 규칙을 하나로 못박고, 십신/용신은 넣지 말 것.
