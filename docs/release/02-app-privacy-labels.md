# 02 · App Store '앱 개인정보(App Privacy)' 라벨 — 영양성분표

> App Store Connect > 앱 개인정보 화면 입력 가이드. 실제 앱이 수집하는 데이터를 기준으로 작성.
> Apple 질문 흐름: (1) 이 데이터를 수집하는가? → (2) 어떤 용도로? → (3) 사용자 계정/신원에 **연결(Linked)** 되는가? → (4) **추적(Tracking)** 에 사용되는가?

---

## 0. 결론 요약

| 항목 | 값 |
|---|---|
| 데이터를 수집하는가 | **예 (Yes, we collect data)** |
| **추적(Tracking)** 에 사용하는가 | **아니요 — 모든 데이터에서 추적 안 함** (제3자 광고·데이터 브로커 공유·기기 간 추적 없음) |
| 제3자 SDK 로그인 | 카카오 로그인 (로그인 목적의 계정 식별에만 사용) |

> '추적(Tracking)'의 Apple 정의: 다른 회사의 앱·웹사이트에서 수집한 데이터와 연결해 타깃 광고·측정에 쓰거나 데이터 브로커에 공유하는 것. 투데이는 이런 행위를 하지 않으므로 **모든 항목 '추적 안 함'.**

---

## 1. 수집 데이터 타입별 표

각 항목: 데이터 타입 / 용도(Purpose) / 계정 연결(Linked) 여부 / 추적(Tracking) 여부.

### 연락처 정보 (Contact Info)

| 데이터 | 수집 | 용도 | 연결됨 | 추적 |
|---|---|---|---|---|
| **이메일 주소 (Email Address)** | 예 | 앱 기능(계정 식별/로그인) | 예 (계정에 연결) | 아니요 |
| **이름 (Name)** — 닉네임 | 예 | 앱 기능·개인화(상대에게 표시할 이름) | 예 (계정에 연결) | 아니요 |

> 이메일·닉네임은 카카오 로그인 및 프로필 설정에서 수집. 별도 전화번호/주소는 수집하지 않음.

### 사용자 콘텐츠 (User Content)

| 데이터 | 수집 | 용도 | 연결됨 | 추적 |
|---|---|---|---|---|
| **사진 (Photos)** | 예 | 앱 기능(일기에 첨부한 사진 저장·상대와 공유) | 예 (계정에 연결) | 아니요 |
| **기타 사용자 콘텐츠 (Other User Content)** | 예 | 앱 기능(일기 본문·오늘의 질문 답장/편지·댓글·반응·장소 별명·기분 이모지) | 예 (계정에 연결) | 아니요 |

### 위치 (Location)

| 데이터 | 수집 | 용도 | 연결됨 | 추적 |
|---|---|---|---|---|
| **대략적 위치 (Coarse Location)** | 예 | 앱 기능(일기에 남긴 장소를 지도에 표시) | 예 (계정에 연결) | 아니요 |

> 설명: 사용자가 일기에 장소를 기록하면 장소명 텍스트와 함께 해당 장소의 좌표(위도/경도)를 저장해 지도에 핀으로 보여준다. 이는 사용자가 **직접 선택/기록한 장소**의 위치이며, 백그라운드 위치 추적이나 이동 경로 수집은 하지 않는다. 지도 시트를 열 때 현재 위치를 지도 중심 표시용으로 일시 사용할 수 있으나(권한 거부 시 서울시청 폴백) 서버에 저장하지 않는다.
> Apple 분류상 '정밀 위치(Precise Location)'가 아닌 **대략적 위치(Coarse Location)** 로 신고. (사용자가 고른 지점 단위이며 실시간 정밀 추적이 아님.)

### 식별자 (Identifiers)

| 데이터 | 수집 | 용도 | 연결됨 | 추적 |
|---|---|---|---|---|
| **사용자 ID (User ID)** | 예 | 앱 기능(계정·커플 연결 식별) | 예 (계정에 연결) | 아니요 |

> 광고 식별자(IDFA)는 수집하지 않음.

### 진단 (Diagnostics) — 선택 항목

| 데이터 | 수집 | 용도 | 연결됨 | 추적 |
|---|---|---|---|---|
| **충돌 데이터 (Crash Data)** | 선택(사용 시) | 앱 기능(안정성 개선) | 아니요 (계정에 연결 안 함) | 아니요 |

> 크래시/성능 분석 도구를 붙이지 않았다면 이 항목은 '수집 안 함'으로 두면 된다. 향후 Sentry/Expo 등 진단 도구를 붙이면 위 표대로 신고한다. **현재 미사용이면 진단 섹션 전체 생략.**

---

## 2. 수집하지 않는 데이터 (참고 — 신고 시 '아니요')

- 금융 정보(결제·카드) — 인앱 결제/구매 없음
- 정밀 위치(Precise Location), 이동 경로, 백그라운드 위치
- 건강·피트니스, 연락처 목록, 검색/브라우징 기록
- 광고 식별자(IDFA), 마케팅/제3자 광고 데이터
- 민감정보(인종·종교·성적 지향 등) — 명시적으로 수집하지 않음

---

## 3. 생년월일·생시(Birthday / Birth time) 처리 메모

- 사용자가 프로필에 입력하는 **생년월일**과 **생시(태어난 시각)** 는 모두 **선택** 항목이며, 기념일/D-day 표시 및 **사주 콘텐츠(운세·궁합)** 산출에 쓰인다. 연결된 커플 상대에게 서비스 내에서 공유된다.
- Apple 라벨에는 '생일/생시' 단독 카테고리가 없다. 가장 가까운 분류는 **기타 사용자 콘텐츠(Other User Content)** 로 보아 위 'User Content'에 포함해 신고한다(민감정보 카테고리 아님, 추적 안 함). 생년월일·생시 둘 다 이 범주로 함께 신고.

---

## 4. 카카오 로그인(제3자 SDK) 메모

- 카카오 로그인은 사용자를 식별하기 위한 인증 수단이다. 이 과정에서 카카오로부터 이메일/닉네임 등 사용자가 동의한 프로필을 받는다.
- 이 데이터는 **로그인/계정 식별** 목적으로만 쓰이며, 광고·추적에 쓰지 않는다.
- 카카오 SDK 자체의 데이터 처리는 카카오의 정책을 따르며, 개인정보 처리방침(`03-privacy-policy.md`)에 제3자 제공/처리위탁으로 명시한다.

## 5. 입력 체크리스트 (App Store Connect에서 순서대로)

- [ ] "데이터를 수집합니다" 선택
- [ ] Email Address — 앱 기능 / 연결됨 / 추적 안 함
- [ ] Name — 앱 기능·개인화 / 연결됨 / 추적 안 함
- [ ] Photos — 앱 기능 / 연결됨 / 추적 안 함
- [ ] Other User Content — 앱 기능 / 연결됨 / 추적 안 함
- [ ] Coarse Location — 앱 기능 / 연결됨 / 추적 안 함
- [ ] User ID — 앱 기능 / 연결됨 / 추적 안 함
- [ ] (진단 도구 사용 시) Crash Data — 앱 기능 / 연결 안 함 / 추적 안 함
- [ ] 모든 항목 "추적에 사용 안 함" 확인
