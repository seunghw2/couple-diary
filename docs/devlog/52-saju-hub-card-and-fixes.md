# 52. 커플 정보 카드(C) + 원국 설명 + 주어 닉네임화

## 요청
- 정보 카드 UI를 C(하트 커플 헤더)로 적용.
- 생시 수정하면 사주 결과도 반영되게.
- 사주 원국이 무슨 뜻인지 화면에서 알려주기.
- 내 사주 화면의 생시 편집은 제거(허브로 일원화).
- (추가) 연인 사주인데 "나"로 나오는 문제 → **닉네임+님**으로. 내 사주·궁합도 일관되게.

## 반영
### 허브 정보 카드 → C (하트 커플 헤더)
- 나 ♥ 연인 아바타(각자 **일간 이모지**: 예 🌊 큰 바다형 / 🌿 넝쿨꽃형)를 마주보게, 아래 각자 생일·생시 미니 박스.
- 내 것(생일·생시)만 탭해서 편집, 연인은 읽기 전용.
- 백엔드 허브 응답에 `myEmoji`/`partnerEmoji`(일간 이모지) 추가.

### 사주 원국 설명
- "사주 자세히 보기" 안 원국의 각 기둥에 **년주/월주/일주/시주** 라벨.
- **"사주 원국이란?"** 접기: 천간·지지, 일간이 나를 대표한다는 점, 기둥별 흐름(초년·사회·나와 짝·말년), 생시로 시주가 채워진다는 설명.

### 생시 편집 위치 정리 + 결과 반영
- 내 사주 화면에서 **생시 편집 그리드 제거**(읽기 전용). 편집은 허브 C카드로 일원화.
- 허브에서 생시 바꾸면 내 사주 재진입 시 새로 불러와 **시주·오행·"오행으로 본 나" 반영**. (일간 캐릭터는 생시와 무관해 그대로 — 명리상 정상)

### "나" → 닉네임+님
- 개인 사주 제목·해설 주어를 닉네임+님으로: "오행으로 본 **요정체리님**", "**요정체리님** 사주엔 목 기운이…". 내 사주도 "근육남친님"으로 일관.
- 개인 사주 렌더를 공용 `PersonalSaju`로 두어 내/연인 동시 적용. 궁합은 이미 실제 닉네임 사용(문제 없음 확인).
- DTO: `PersonalResult.ownerName` 추가, `ohaengInsight(ec, subject)`로 주어 파라미터화.

## 검증
- 백엔드 clean compile / 프론트 tsc 0.
- 운영 백엔드 스모크: 허브 myEmoji 🌊·partnerEmoji 🌿, me/partner 해설 주어 "근육남친님/요정체리님" 정상.
