# 14 · 일기 작성 복귀 백지/리셋 버그 — 초안 자동복원

**날짜**: 2026-07-05
**증상**: 일기 작성 중 다른 앱 갔다가 돌아오면 화면이 백지가 되거나 처음(모드 선택)으로 리셋되고, 쓰던 내용이 사라짐.

## 원인
iOS(Expo Go)에서 백그라운드 복귀 시 앱이 **리마운트**(번들 리로드/메모리 회수)되면서 작성 화면의 React 상태(step·기분·별점·답·장소·사진)가 통째로 초기화됨. 기존 일기가 있는 날짜면 `loadingDetail` 스피너 단계로 돌아가 **헤더도 없는 백지**로 보임. (웹에선 재현 안 됨 → 기기 리마운트 특성)

## 수정
React 상태는 리마운트를 못 버티므로 **작성 초안을 기기 저장소에 보관**:
- `lib/writeDraft.ts`: `write_draft:{date}` 키로 AsyncStorage에 초안 저장/복원/삭제.
- 작성 화면: `form` 단계에서 내용 변경 시 **debounce(600ms) 저장**, 진입(서버 detail 로딩 후) 시 초안 있으면 **복원**, 저장 성공하면 초안 삭제. 빈 초안은 무시.

## 검증
웹 전체 리로드(=리마운트)로 재현·확인:
- 입력 → 초안 저장(localStorage) → 리로드 → **폼 상태 + 입력 내용 그대로 복원**(모드선택/백지로 안 돌아감).
