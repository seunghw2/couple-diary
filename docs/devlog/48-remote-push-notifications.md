# 48. 원격 푸시 알림 (앱 꺼져 있어도 도착)

## 요청
- "알림 오면 앱 꺼져있을 때도 알림 오게 개발되어있어? EAS 빌드한 앱에서도 알림 권한 허용도 안 했고 실제로도 안 옴."

## 현황(작업 전)
- 인앱 알림은 완비(서버 DB + 폴링)돼 있었지만 **원격 푸시는 미구현**.
  - 알림 권한 요청 없음 → 권한 프롬프트가 뜬 적 없음
  - 푸시 토큰 등록/발송 코드 없음
  - 원래 Expo Go라 원격 푸시가 막혀 "EAS 후속"으로 미뤄둔 상태

## 반영 — 진짜 폰 푸시 전체 구현
앱이 꺼져 있어도 잠금화면에 알림이 오도록 Expo Push 기반으로 붙였다.

**백엔드**
- `push_tokens` 테이블 + 등록/해제 API (`POST/DELETE /api/push-tokens`)
- `PushSender`: 알림이 생길 때 수신자의 모든 기기에 Expo Push API로 발송(@Async — 알림 저장을 네트워크로 붙잡지 않음, 실패해도 앱 흐름 무영향)
- 기존 14종 알림(일기·댓글·질문·월드컵·기념일·콕찌르기…) 저장 경로를 모두 push 발송과 연결
- 유효하지 않은 토큰(앱 삭제 등)은 발송 응답 보고 자동 정리
- 계정 삭제 시 푸시 토큰도 함께 삭제

**프론트**
- expo-notifications/expo-device 도입, app.json 플러그인 등록
- 로그인 후 **알림 권한 요청 + 푸시 토큰 발급 → 서버 등록**(세션당 1회)
- 알림 **탭 시 해당 화면으로 딥링크**(월드컵 결과/그날 일기), 수신 시 인앱 목록 최신화
- 포그라운드에서도 배너/소리 노출

## 검증
- 백엔드 clean compile / 프론트 tsc 0
- 운영 백엔드에 유효 JWT로 등록→업서트(1행 유지)→해제(0행) 왕복 확인, `push_tokens` 자동 생성 확인

## ⚠️ 실기기 반영 조건
- 원격 푸시는 **EAS 빌드 + 실제 기기에서만** 동작(Expo Go/시뮬레이터는 권한 프롬프트까지만).
- iOS 푸시 키(APNs)는 다음 EAS 빌드 때 EAS가 자동 설정.
- 즉, **다음 EAS 빌드 후** 앱 꺼져 있어도 알림이 온다.
