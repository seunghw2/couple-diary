# 49. 사주 궁합 결과 재구성 (신뢰감·구체성·구조·공유)

## 요청
- 사주 결과지 피드백: "예쁘고 읽기 편한데 아직 너무 무난한 결과지." 서브에이전트 4개로 분석 후 재구성.
- 핵심 3대 목표: ① 점수와 설명이 논리적으로 연결 ② 누구에게나 맞는 말 대신 구체적 커플 행동 ③ 읽는 결과지가 아니라 공유·대화하게 만드는 결과.

## 분석
- 4개 서브에이전트 병렬 분석 → 종합 기획서 `docs/saju/04-결과화면-재구성-기획.md`.
- 공통 발견 2가지:
  - **막대 색 버그**: 프론트가 `grade>=4/3`을 검사했는데 백엔드 grade는 0/1/2뿐 → 사실상 노랑·갈색 2색만. "대화만 노랑, 나머지 갈색"의 원인.
  - **재미 부족의 근본**: 계산 엔진은 일지충·상보·화 부족 등을 다 뽑는데, 문장은 등급(0/1/2)만 보고 골라 그 정보를 버림 → 다 "다독여주는/채워주는"으로 수렴.

## 반영

### 백엔드 (라이브 배포 완료)
- **feature → 행동 시그니처**: 카테고리마다 가장 강한 관계 feature 하나를 골라 "행동 장면" 한 문장을 덧붙임(닉네임 포함). 예 "근육남친님은 마음이 정해지면 바로 움직이고, 체리회장님은 한 박자 살핀 뒤 확신이 서요."
- **점수-스토리 총평**: 최고 카테고리 칭찬 + 최저 카테고리 '차이'(성장 프레임)를 한 문장으로 엮음. 예 "말이 놀랄 만큼 잘 통하는데 생활 리듬은 아직 맞춰가는 중이라, 시간이 편이 되어줄 사이예요."
- **대표 한줄**(히어로): 가장 희귀·강한 signal 하나. 예 "다 챙기면서 '좋아해'만 아끼는, 표현이 숙제인 사이예요."
- **딱 3줄 해석 / 관계 키워드 / 최저 카테고리** 신규 산출.
- **관계 꿀팁**: 무작위 → 최저·차저 카테고리 처방 + 최고 카테고리 유지팁(왜+뭘 구조).
- **등급 컷 상향**(67/34 → 70/45): 47점 애정이 '중'에 머물러 온도가 어긋나던 문제 완화.
- **본문 이모지 제거**: 카테고리·총평·대표한줄·꿀팁 문구 끝 이모지 정리(문단마다 붙던 난립 제거). 배지는 유지.
- DTO 확장: `weakestKey`, `keywords`, `summaryLines`, `meNickname`/`meTypeName`/`partnerTypeName`.

### 프론트
- **점수 구간색·라벨**: 80+ 골드/찰떡 · 60~79 브라운/안정적 · 0~59 로즈/맞춰가기 (막대 색 버그 수정 겸). 히어로 %에도 구간색.
- **화면 재구성**: 대표결과(이름+유형명+대표한줄+%) → 우리 관계 한눈에(최고/최저/키워드) → 딱 3줄 해석 → 항목별 궁합(2줄 프리뷰 + '자세히 보기' 아코디언) → 관계 꿀팁 → 사주 근거 보기(접힘) → 공유.
- **종합 점수 ⓘ**: 종합과 항목 평균이 왜 다른지 설명 모달(신뢰감).
- **공유 버튼**: soft(비활성처럼) → primary + 하단 고정, 문구 "우리 궁합 카드 공유하기".
- **이모지 통일**: 카테고리당 대표 아이콘 1개(첫끌림✨·대화💬·애정♥·안정감🏠·성장🌱).
- 아코디언은 RN 순정 `LayoutAnimation`(새 의존성 0), 공용 `components/Collapsible.tsx` 신설.
- 개인 사주(me)도 원국·생시를 "사주 자세히 보기" 접기로 정리.

## 검증
- 백엔드 clean compile / 프론트 tsc 0.
- 운영 백엔드에 유효 JWT(커플 19/20)로 `/api/saju/couple` 호출 → 유형명·키워드·대표한줄·엮은 총평·약점연결 꿀팁·feature 시그니처·이모지 제거 모두 정상. 닉네임 중복 "님님" 교정 확인.
