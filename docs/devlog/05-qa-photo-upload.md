# 05 · QA + 버그 수정 + 실제 사진 업로드

**날짜**: 2026-07-05
**목표**: 3자 테스터처럼 QA해서 버그 잡고, 오늘 밤 커플 실사용에 필요한 미구현 기능 구현.

## QA (서브에이전트 4개 병렬)
- 기능 테스터(Expo Web + Playwright 2인 플로우), 백엔드 감사, 프론트 UI/UX 감사, 커플 사용 기능갭 분석.
- **P0 크래시 0개**(핵심 루프 정상), 다수 P1/P2 발견.

## 확인된 사실
- **사진**: `photos` 테이블에 실제 이미지 아닌 `colorSeed` 문자열만 저장 → 진짜 사진 업로드 미구현(핵심 갭).
- MySQL 데이터(유저·커플·일기·답·댓글)는 정상 저장 중.

## 수정 (백엔드/프론트 병렬)
- 백엔드: DiaryDay **동시 첫 작성 경합**(get-or-create), **부분 수정 데이터 유실**(null=유지/빈배열=삭제), 질문픽 범위 검증, 입력 검증(@Min/@Max/@Size), 미래 날짜 차단, 예외→400 매핑, 커플 유니크, **일기 삭제(DELETE)**.
- 프론트: 401 전역 로그인 복귀, 질문픽 폴백 NaN 제거, **상세 질문문장 표시**, **캘린더 스티커 위치 버그**, D-day 음수, 댓글 실패 표시, 기념일 마스킹 입력, 닉네임 수정, **수정 프리필**, 일기 삭제, "상대가 기다려요" 배너, placeholder 대비.

## 🖼️ 실제 사진 업로드 구현
- `POST /api/photos`(multipart) → 로컬 디스크 `~/couple-diary-uploads`에 UUID 저장, `/files/**` 정적 서빙, `Photo.url` 컬럼.
- 프론트: `expo-image-picker`로 갤러리 선택 → 업로드 → `photoUrls`로 전송, 상세/캘린더에 실제 이미지 렌더.
