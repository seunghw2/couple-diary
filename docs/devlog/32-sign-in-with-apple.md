# 32. Sign in with Apple 구현

앱스토어 심사에서 가장 큰 반려 위험이던 **Apple 로그인**(가이드라인 4.8)을 붙였다. 소셜 로그인(카카오)이 있으면 애플 로그인도 동등하게 제공해야 한다.

## 사용자가 보는 것
- 로그인 화면 **맨 위에 검은색 'Apple로 로그인' 버튼** 추가(카카오 위). iOS에서만 보인다.
- 버튼을 누르면 시스템 Apple 로그인 시트 → Face ID/암호로 인증 → 바로 로그인. 최초 1회는 이름/이메일 제공 여부를 고를 수 있다.
- 이후 흐름(커플 연결 등)은 카카오와 동일.

## 동작 방식
- 앱: `expo-apple-authentication`로 Apple에서 서명된 `identityToken`을 받아 서버로 전달.
- 서버: `POST /api/auth/apple`
  - Apple 공개키(JWKS)로 토큰 **서명·발급자·대상(우리 번들 ID)·만료**를 검증.
  - 통과하면 Apple 고유 ID(`sub`)로 유저를 찾거나 새로 만들고 우리 JWT 발급.
  - 카카오와 완전히 같은 응답(`accessToken` + 유저 요약)이라 이후 로직 공유.
- 유저 테이블에 `apple_id`(유니크) 컬럼 추가. 이메일 미제공 시 `apple_<sub>@today.local`로 대체(카카오와 동일 패턴).

## QA
- 백엔드 컴파일 0, 부팅 0 에러, `apple_id` 컬럼+유니크 인덱스 자동 생성 확인.
- `POST /api/auth/apple`에 잘못된 토큰 → **401 A001**(검증 경로 정상 동작).
- 프론트 `tsc` 0.
- 실제 로그인 성공 흐름은 **실기기/EAS 빌드에서만** 검증 가능(아래).

## 알아둘 점 (중요)
- Apple 로그인은 **Expo Go에서 동작하지 않는다.** 시스템 엔타이틀먼트가 필요해 **EAS 개발 빌드 또는 프로덕션 빌드**에서만 뜬다. 지금 Expo Go로 열면 iOS라도 버튼 눌렀을 때 실패한다.
- **App Store Connect / 개발자 콘솔 설정 필요:** App ID에 "Sign In with Apple" Capability 켜기 + 프로비저닝. EAS 빌드가 자동 처리하지만 최초 1회 계정 연결이 필요하다.
- **계정 삭제 시 토큰 revoke(5.1.1v)는 후속 작업:** Apple은 SIWA 계정 삭제 시 토큰 폐기를 권장한다. 이건 Apple `.p8` 비공개키 + authorizationCode 교환이 필요해, 키 발급 후 별도로 붙일 예정. (현재 계정 삭제는 우리 DB 데이터는 완전히 지운다)
