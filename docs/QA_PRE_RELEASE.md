# 출시 전 제3자 QA 리포트 (2026-07-16)

## 방법
- 서브에이전트 5개가 **각자 격리된 테스트 커플(계정 2개)** 을 만들어 도메인별로 API를 철저히 테스트.
  (prod 프로파일이라 dev-login 비활성 → DB에 QA 커플 provisioning + JWT 발급, 테스트 후 전량 삭제)
- 도메인: ①계정·인증·커플연결·프로필 ②일기·댓글·캘린더·지도 ③오늘의 질문 ④사주·월드컵 ⑤알림·콕·의견·푸시.
- 정상흐름 + 엣지케이스 + 입력검증 + 권한/격리까지 총 200+ 케이스.

## 총평
핵심 로직은 전반적으로 **견고**하다. 특히:
- 일기 **잠금/공개** 로직, 오늘의 질문 **상대 답 가림**: 데이터 누출 없음(양쪽 봉인 전엔 상대 답 미노출).
- 입력 검증(길이·범위·포맷·날짜)과 권한 격리(남의 알림/일기 접근 차단)는 대부분 정확.
- 사주 값(십성·궁합 sajuNote·운세 점수 밴드·한자 표기)·월드컵 비교 정상.

출시를 막을 **High 4건 + Medium 4건**을 발견해 **모두 수정·재검증 완료**했다.

## 수정 완료 (High/Medium)

| # | 심각도 | 문제 | 수정 | 재검증 |
|---|---|---|---|---|
| 1 | High | 인증 실패(401/403) 응답 한글 깨짐(charset 미지정 → `????`) | SecurityConfig entryPoint/accessDeniedHandler에 UTF-8 인코딩 지정 | `로그인이 필요해요.` 정상 |
| 2 | High | 알림설정 부분 PUT 시 미지정 필드가 조용히 전부 false로 꺼짐 | 필드를 Boolean(nullable)로, null이면 기존값 유지(PATCH 시맨틱) | `{}` PUT → 전부 유지 |
| 3 | High | 질문 신고(report)가 커플에 배정된 적 없는 임의 질문도 신고 가능 | 배정 이력(daily_questions) 소속 검증 추가 | 미배정 질문 신고 차단 |
| 4 | High | 첫 작성 검증 실패 시 빈 고아 날짜가 캘린더에 영구 잔존 | 월 응답에서 글 0개인 빈 날 제외(고아·삭제 잔여 숨김) | 고아 날짜 0개 |
| 5 | Medium | 잘못된 HTTP 메서드 → 500 (catch-all이 삼킴) | 405 핸들러 추가 | 405 반환 |
| 6 | Medium | 기념일(처음 만난 날) 미래 날짜 허용 → D-day 음수 | 미래 날짜 400 가드(생일과 대칭) | 400 차단 |
| 7 | Medium | 오늘의 운세 배너 색/키워드와 상세 화면 값 불일치(출처 상이) | 배너를 상세와 동일 소스(SajuDailyFortune)로 단일화 | 배너=상세 일치 |
| 8 | Medium | 월드컵 결과에서 우승자가 탈락 목록에 동시 포함돼도 저장 | 우승자가 탈락에 있으면 400 거부 | 400 차단 |
| + | Low | feedback source 검증 메시지가 Bean Validation 기본 문구 | 커스텀 메시지 지정 | — |

## 보류(문서화, 이번 미수정) — 정책/경미

- **닉네임 HTML 미필터**(저장형 XSS 여지): 프론트가 파트너 닉네임을 이스케이프 렌더하는지 확인 필요. 30자 제한으로 영향 제한적. → 프론트 렌더 점검 권장.
- **삭제→재작성 시 옛 댓글 부활**(한쪽만 삭제 시): 정책 결정 필요("일기 지웠는데 옛 댓글이 되살아남" 혼란 가능).
- **OPENED 후 답장 무음 수정**(24h 이내): 열람된 편지 수정 잠금/'수정됨' 표시 검토.
- **콕 쿨다운이 수신자 기준**(발신자 무관): 2인 커플에선 무해. 다자 확장 시만 문제.
- **월드컵 결과 중복 누적**(dedup 없음): 히스토리 의도면 정상, 재플레이=덮어쓰기면 로직 필요.
- **생시 빈 바디 PUT이 생시 삭제**: 오삭제 방어 검토.
- **커플 percent vs 카테고리 평균 스케일 상이**: 버그 아님(이미 안내 모달 있음).
- **미연결 콕 상태코드 400/404 혼재**: COUPLE_NOT_FOUND(404)로 통일 권장.
- **개인 사주 서사가 일간 기준**: 십성 추가로 부분 해소됨. 완전 개인화는 후속.

## 정리
QA용 계정·커플·데이터는 테스트 후 전량 삭제(사용자 수 원복 확인). 실사용 데이터·통계 영향 없음.
