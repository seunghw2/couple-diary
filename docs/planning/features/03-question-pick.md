# 03 · 자유 일기 — 질문 픽 모드

## 목적
고정 템플릿의 답답함을 풀되, 완전 백지의 부담도 없게. **먼저 쓰는 사람이 그날의 질문을 고르면 둘이 같은 질문에 답하는** 커플만의 문답 일기. 질문을 고르는 행위 자체가 "오늘 너에게 묻고 싶은 것"이라는 메시지가 된다.

## 동작 (핵심 플로우)
1. 작성 화면에서 템플릿 칩 중 **자유** 선택
2. **기본 질문 리스트**(아래)에서 **3개 선택** → 이 3개가 그날의 "오늘의 질문"으로 **커플 공통 고정**
3. 먼저 쓴 사람(A)은 고른 3개에 답해 저장 (잠금 대기)
4. 나중에 쓰는 사람(B)에겐 "A님이 오늘의 질문 3개를 골랐어요 💌" — **그 3개 범위 안에서** 답하고 싶은 질문을 골라(최소 1개) 답함
5. 둘 다 저장하면 공개 — 상세에서 **질문별로 두 사람 답이 나란히** 비교됨

## 기본 질문 리스트 (v1, 8개)
1. 기억에 남는 장면 2~3가지
2. 오늘 내가 느낀 감정 한 줄
3. 오늘 우리가 잘 맞았던 순간
4. 한 줄 요약 — 오늘은 "___"한 하루였다 (인라인 빈칸형)
5. 고마웠던 순간
6. 상대가 오늘 귀여웠던 순간
7. 오늘 살짝 아쉬웠던 것
8. 다음에 우리 같이 하고 싶은 것

> 이후 확장: 질문 팩(계절/여행/싸운 날), 커스텀 질문 추가, 자주 쓴 질문 상단 정렬.

## 데이터
- `entry.mode`: `template` | `question_pick`
- `dailyQuestionSet { date, coupleId, questionIds[3], pickedBy }` — 하루에 하나, 먼저 쓴 사람이 생성
- `entryAnswer { entryId, questionId, text }` — B는 1~3개만 있을 수 있음
- 질문 마스터: `question { id, text, type: normal|inline_blank }`

## 규칙 정합성
- 질문 세트는 **그날 첫 작성자만** 고를 수 있음 (B 진입 시 이미 고정)
- 상호 공개·수정 3시간 지연·댓글 규칙은 일반 일기와 동일
- B가 답 안 한 질문은 상세에서 "아직 답이 없어요"로 표시(비난 느낌 없이)

## 화면
- 질문 고르기(3/3 카운터) → 질문별 답 작성 → (상대) 오늘의 질문 안내+범위 내 선택 답변
- 목업: `../assets/03-full-app/mockups/write-free/` · 스토리보드 [03 문서 3장](../03-full-storyboard.md)

## 엣지케이스
- 동시에 둘 다 첫 작성 진입 → 먼저 저장한 쪽 세트가 채택, 늦은 쪽엔 "질문이 정해졌어요" 안내 후 답변 화면으로
- A가 저장 전 이탈 → 세트 미확정(임시저장만)
- 질문 3개 미만 선택 시 "쓰기" 버튼 비활성
- B가 0개 답변 저장 시도 → 최소 1개 안내
