# 25 · 오늘의 질문 — AI 질문 세트 설계

> [24 구현안](24-daily-question-v1-spec.md)의 `question_pool`을 **AI로 어떻게 구성·운영할지**를 5단계로 구체화한 기획.
> (화면·스키마는 24에서 확정. 이 문서는 '어떤 질문이, 어떻게 나오는가'.)

## 5단계 결정 요약
| 단계 | 결정 |
|---|---|
| 1. 핵심 목적 | 관계심화 : 일상 : 재미 = **1 : 3 : 1** (가벼운 일상 우세) |
| 2. 분류 체계 | **테마(주제) 중심** + 객관 데이터 기반 **맥락 질문 가끔** |
| 3. 깊이 진행 | **리듬 배합** (평소 가벼움, 주말·기념일·주기엔 깊거나 특별) |
| 4. 개인화 | **가벼운 맥락** (기념일·D-day·생일·계절·요일·연속일 등 객관 신호만, 일기 내용은 안 읽음) |
| 5. 생성·운영 | **주기 배치 보강** (시드 풀 + 주기 AI 배치 생성·자동 필터·'별로예요' 피드백) |

---

## 1. 톤 비율 (1 : 3 : 1)
각 질문은 `tone` 하나를 가진다. 장기적으로 배정 비율이 아래로 수렴하도록 가중.
- **deep(관계심화) 20%** — 마음·미래·관계 돌아보기. 뭉클·진지.
- **light(일상) 60%** — 오늘 하루·취향·소소한 추억. 부담 없이 매일.
- **fun(재미) 20%** — 상상·유머·밸런스. 톡톡.
- (별도 `context`(맥락/기념)는 트리거 시 위 비율과 무관하게 우선 삽입 — §4)

## 2. 테마 분류 (taxonomy)
`theme` = 질문의 주제. 각 테마는 기본 tone 성향을 갖되 예외 허용.
| theme | 성향 | 예 |
|---|---|---|
| 오늘/일상 | light | "오늘 하루 중 제일 좋았던 순간은?" |
| 취향/선호 | light | "요즘 제일 자주 듣는 노래는?" |
| 추억/함께한 시간 | light~deep | "우리 첫 데이트에서 제일 기억나는 장면은?" |
| 마음/애정표현 | deep | "요즘 나한테 가장 고마웠던 순간은?" |
| 미래/약속 | deep | "10년 뒤 우리는 어디서 뭘 하고 있을까?" |
| 관계 돌아보기 | deep | "우리 사이에서 제일 소중한 게 뭐라고 생각해?" |
| 상상/놀이 | fun | "복권 당첨되면 둘이 제일 먼저 뭐 할까?" |
| 맥락/기념 | context | (템플릿, §4) |

## 3. 리듬 배합 (요일·주기 기반)
개인 일기 내용을 안 보고도 '완급'을 만드는 규칙.
- **월~목(평일)**: light 우세 + fun 가끔. (가볍게, 매일 습관화)
- **금~일(주말)**: deep 비중↑ (마음·미래·추억). 여유 있게 곱씹는 편지.
- **기념일·생일·마일스톤**: `context` 질문을 **우선 삽입**(§4).
- 매일 오는 **2개 후보는 서로 다른 tone/theme**로 구성(고르는 재미).
- 장기적으로 §1의 1:3:1이 유지되도록, 리듬은 '그날의 목표 분포'만 바꾸고 전역 비율은 가중 보정.

## 4. 가벼운 맥락 (context triggers)
객관·비침습 신호만 사용. 트리거 활성 시 그날 2개 중 1개를 맥락 질문(템플릿 렌더)으로.
| 트리거 | 조건 | 질문 예(템플릿) |
|---|---|---|
| 기념일 임박/당일 | N주년·100·200일 D-3~D0 | "곧 {N}일이야. 그날 뭐 하고 싶어?" / "{N}주년인 지금, 제일 고마운 건?" |
| 생일 | 본인/상대 생일 D-3~D0 | "{상대}야, 이번 생일에 제일 받고 싶은 하루는?" |
| 계절/명절 | 봄·여름·크리스마스 등 | "올여름 우리 꼭 같이 하고 싶은 건?" |
| 연속 마일스톤 | 편지 7·30·100일째 | "벌써 편지 {N}통째야. 요즘 우리 어때?" |
| 첫 편지 / 오랜만 복귀 | 첫날 / 공백 후 재개 | "다시 편지 시작! 요즘 네 하루는 어때?" |
- 우선순위: 기념일·생일 > 마일스톤 > 계절. 여러 개면 하나만.
- 템플릿 placeholder(`{N}`,`{상대}`)는 커플의 객관 데이터로 서버가 치환. **일기 본문·기분은 사용 안 함.**

## 5. 생성·운영 파이프라인 (주기 배치 보강)
### (a) 시드 풀
초기 큐레이션된 수십~수백 문항(현재 코드 시드 30개 확장). `source='seed'`.

### (b) 주기 배치 생성
- **주기**: 월 1회 또는 활성 풀이 임계 이하로 소진될 때.
- **생성 프롬프트(개요)**: 시스템에 앱 정체성 주입 — "커플 교환일기 '오늘의 질문'. **반말·따뜻·편지 감성**. 두 사람이 각자 답하고 서로 공개하는 구조." + 요청: `theme`·`tone`·개수 지정해 JSON 배열 생성. 출력 필드: `{text, theme, tone, depth, contextTrigger?}`.
- **금지 가이드**: 과한 성적 표현, 정치·특정 종교 우열, 비난·불쾌 유도, 답이 곤란한 사생활 추궁, 특정 정답이 있는 상식퀴즈. 물음표로 끝나는 열린 질문만.

### (c) 자동 필터(게이트)
배치 생성물은 자동 검사 후 통과분만 `active=true`, 경계선은 검수 큐(수동).
1. **중복/유사도**: 기존 풀과 임베딩 코사인 유사도 > 임계면 제거.
2. **형식**: 길이(8~60자 권장), 물음표 종결, 반말 톤.
3. **민감어/금칙**: 금지 가이드 위반 키워드 차단.
4. **밸런스**: theme/tone 목표 분포에 맞춰 채택(과다 테마 컷).

### (d) 피드백 루프
- 사용자 '이 질문 별로예요' 신고 → `reportedCount++`.
- 임계(예: 서로 다른 커플 3팀 이상 or 노출 대비 신고율) 초과 → **자동 `active=false`** + 재검토 큐.
- `usedCount`·오픈율(둘 다 답한 비율)로 '좋은 질문' 랭킹 → 배정 가중에 반영(선택).

### (e) 스키마 확장 (24의 `question_pool`에 추가)
```
question_pool
  ... (기존: id, text, active, source, createdAt)
  theme         enum(오늘/취향/추억/마음/미래/관계/놀이/맥락)
  tone          enum(deep|light|fun|context)
  depth         int(1~3)          -- 리듬/난이도 보조
  contextTrigger enum? (anniversary|birthday|season|streak|firstletter|comeback)
  isTemplate    boolean           -- placeholder 포함 맥락 질문 여부
  usedCount     int default 0
  reportedCount int default 0
  openRate      float?            -- 품질 지표(선택)
  batchId       varchar?          -- 생성 배치 추적
```

## 6. 매일 2개 배정 알고리즘(종합)
1. 그날 **컨텍스트 신호** 계산(D-day·생일·요일·연속일).
2. 트리거 활성 → 슬롯1을 `context` 질문(템플릿 렌더)으로 우선.
3. 나머지 슬롯 → **요일 리듬의 목표 tone 분포**에서, 최근 N일(예 45일) 사용 회피 + 전역 1:3:1 보정 가중 랜덤으로 pick.
4. 2개의 tone/theme가 겹치지 않게. (예: context 1 + light 1, 또는 light 1 + deep 1)
5. 활성 풀 부족 시 배치 생성 트리거.

## 7. 착수 순서(제안)
1. 스키마 확장(theme/tone/contextTrigger/isTemplate/카운터) + 시드 재분류·확장.
2. 배정 알고리즘(리듬+톤비율+컨텍스트+최근회피) 적용.
3. 맥락 트리거·템플릿 렌더(기념일·생일·마일스톤).
4. '별로예요' 신고 API + 자동 비활성 임계.
5. 배치 생성 파이프라인(프롬프트·자동 필터·검수 큐) — 운영 도구.

> 프라이버시 원칙: **일기 본문·기분 데이터는 질문 생성/배정에 사용하지 않는다.** 오직 기념일·생일·요일·연속일 등 객관 신호만.
